For example, if you enter `example<loopRepeat>` in the value field and the current loop repeat is `5`, the resolved value becomes `example4`.

## Supported Patterns

<BsTable>

| **Pattern**                       | **Description**                                                |
| --------------------------------- | -------------------------------------------------------------- |
| [`<loopRepeat>`](#looprepeat)     | Replaces with the current loop repeat index (starting from 1). |
| [`<stepRepeat>`](#steprepeat)     | Replaces with the current step repeat index (starting from 1). |
| [`<sessionCount>`](#sessioncount) | Replaces with the current session count (starting from 1).     |
| `<batchRepeat>`                   | Legacy alias for `<loopRepeat>`.                               |
| `<actionRepeat>`                  | Legacy alias for `<stepRepeat>`.                               |

</BsTable>

### `<loopRepeat>`

`example<loopRepeat>`

<Code code={`example<loopRepeat>  → example1`} lang="html" />

### `<stepRepeat>`

`item<stepRepeat>`

<Code code={`item<stepRepeat>  → item1`} lang="html" />

### `<sessionCount>`

`run<sessionCount>`

<Code code={`run<sessionCount>  → run1`} lang="html" />

<Callout type="info">
  Older placeholders still work for backward compatibility: use `<batchRepeat>` as a legacy alias for
  `<loopRepeat>`, and `<actionRepeat>` as a legacy alias for `<stepRepeat>`.
</Callout>
